## Screenshot Guidelines

When adding screenshots, please follow these guidelines:

### File Naming Convention

```
[feature]-[device]-[number].png

Examples:
- command-suggestions-desktop-1.png
- options-page-desktop-1.png
- mobile-interface-phone-1.png
- game-integration-desktop-1.png
```

### Recommended Screenshots

1. **Extension in Action** (`game-integration-*.png`)
   - Show command suggestions appearing in a game
   - Highlight the AI suggestions below the input field
   - Capture during an interesting moment in gameplay

2. **Options Page** (`options-page-*.png`)
   - Full view of the extension options/settings page
   - Show both Ollama and Gemini configuration
   - Include all major settings

3. **Mobile Interface** (`mobile-interface-*.png`)
   - Touch-friendly buttons on a mobile device
   - Demonstrate responsiveness
   - Show ease of use on smaller screens

4. **Command Palette** (`command-palette-*.png`)
   - Close-up of suggestion buttons
   - Show keyboard shortcuts indicators
   - Display different types of suggestions

5. **Setup Process** (`setup-*.png`)
   - Installation in Chrome
   - First-time configuration
   - Connection testing

### Image Specifications

- **Format**: PNG (preferred) or JPG
- **Desktop screenshots**: 1280x800 or 1920x1080
- **Mobile screenshots**: 375x812 (iPhone) or 360x640 (Android)
- **Chrome Web Store screenshots**: 1280x800 or 640x400
- **File size**: < 2MB per image
- **Quality**: High resolution, clear text

### Privacy Considerations

When taking screenshots:

- ✅ Use publicly available games
- ✅ Remove any personal information
- ✅ Hide API keys in options screenshots
- ❌ Don't include copyrighted game content (unless permission obtained)
- ❌ Don't show identifying information

### Processing Screenshots

Before committing:

1. Optimize file size (use tools like TinyPNG)
2. Add descriptive alt text in documentation
3. Verify screenshots render correctly in README.md
4. Check that screenshots are up-to-date with current UI

### Using Screenshots

Screenshots from this directory are used in:

- `README.md` - Main project overview
- `docs/INSTALL.md` - Installation guide
- Chrome Web Store listing
- Social media and announcements
- Blog posts and articles

## Contributing Screenshots

To contribute screenshots:

1. Capture high-quality screenshots following the guidelines above
2. Name files according to the convention
3. Optimize images for web
4. Add to this directory
5. Update relevant documentation to reference the new screenshots
6. Submit a Pull Request
